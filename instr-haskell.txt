# Alternatively, to install on your machine, we suggest
  installing a package manager for Haskell, called Stack

Installation instructions at https://docs.haskellstack.org/en/stable/install_and_upgrade/.

In summary, for Ubuntu, run `curl -sSL https://get.haskellstack.org/ | sh`.
To upgrade the system installation, run `curl -sSL https://get.haskellstack.org/ | sh -s - -f`

## Using the system GHC for all users when possible

To force usage of the system GHC if it exists, add the following to `/etc/stack/config.yaml`:

```
system-ghc: true
```

This will use the system GHC if it matches the required GHC for a project.

# Installing system GHC (Ubuntu)

To install a system level GHC, run the following commands to add a repository with updated GHC versions:

```
sudo add-apt-repository ppa:hvr/ghc
sudo apt-get update
```

To install GHC, run:

```
sudo apt-get install ghc-<version> ghc-<version>-dyn
```

where <version> is replaced by the GHC version you want to install.

Because this repo does not install to /usr (to avoid interfering with Ubuntu's packages), we need to add `/opt/ghc/bin`
to the PATH, e.g., adding `export PATH="/opt/ghc/bin:$PATH"` to `.bashrc`.

# Emacs integration

Detailed instructions here: https://github.com/serras/emacs-haskell-tutorial/blob/master/tutorial.md

Simple summary:

- Install ghc-mod globally (run `stack install ghc-mod` in home folder)
- Install `haskell-mode`, `company-ghc` packages in emacs
- Add the following configuration to emacs's init:

```
(autoload 'ghc-init "ghc" nil t)
(autoload 'ghc-debug "ghc" nil t)
(add-hook 'haskell-mode-hook (lambda () (ghc-init)))
(require 'company)
(add-hook 'haskell-mode-hook 'company-mode)
(add-to-list 'company-backends 'company-ghc)
```

The following belongs to a `custom-set-variables` section. If such a section already exists in your init, just
add the clauses. Otherwise, feel free to copy the whole section.

```
(custom-set-variables
  '(company-ghc-show-info t)
  '(haskell-process-type 'stack-ghci)
)
```
